## Business Glossary

### Sanctioned Counterparty
Definition: A counterparty with is_sanctioned = TRUE
Formula: SELECT * FROM counterparties WHERE is_sanctioned = TRUE
Source: counterparties
Unit: Boolean

### Settlement Date
Definition: The date when a trade is settled
Formula: trade_date + INTERVAL '1 day'
Source: trades
Unit: Date

### Value-at-Risk (VaR)
Definition: The maximum potential loss of a portfolio over a given time horizon with a given probability
Formula: SELECT var_95 FROM risk_metrics
Source: risk_metrics
Unit: Currency

### Revenue
Definition: The total profit earned from executed trades
Formula: SELECT SUM(total_pnl) FROM daily_pnl WHERE status = 'EXECUTED' AND trade_date >= fiscal_year_start
Source: daily_pnl
Unit: Currency

### Exposure
Definition: The total notional value of a portfolio
Formula: SELECT SUM(notional_amount) FROM trades WHERE counterparty.country IN ('IR', 'KP', 'SY', 'CU') AND notional_amount > 500000
Source: trades
Unit: Currency

## Entity Relationships

- accounts --[one-to-many]--> positions (via account_id)
  An account can have multiple positions
- counterparties --[many-to-one]--> trades (via counterparty_id)
  A counterparty can be involved in multiple trades
- instruments --[many-to-one]--> trades (via instrument_id)
  An instrument can be traded multiple times
- positions --[one-to-one]--> risk_metrics (via position_id)
  A position has one corresponding risk metric

## Business Rules (MUST enforce)

### [CRITICAL] Sanctioned Counterparty Check
No trade may be executed with a counterparty where is_sanctioned = TRUE
SQL patterns:
  SELECT * FROM trades WHERE counterparty_id NOT IN (SELECT counterparty_id FROM counterparties WHERE is_sanctioned = TRUE)
  INSERT INTO trades (trade_date, settlement_date, counterparty_id, instrument_id, trade_type, quantity, price, notional_amount, currency, status) VALUES (:trade_date, :settlement_date, :counterparty_id, :instrument_id, :trade_type, :quantity, :price, :notional_amount, :currency, :status) WHERE :counterparty_id NOT IN (SELECT counterparty_id FROM counterparties WHERE is_sanctioned = TRUE)
  severity_level: CRITICAL
  source_tables: trades, counterparties
  source_columns: counterparty_id, is_sanctioned
  unit: Boolean
  synonyms: Blocked Counterparty, Prohibited Counterparty
  description: This rule ensures that no trade is executed with a sanctioned counterparty.

### [HIGH] Settlement Date Validation
Settlement date must be >= trade_date (T+1 or later)
SQL patterns:
  SELECT * FROM trades WHERE settlement_date >= trade_date + INTERVAL '1 day'
  INSERT INTO trades (trade_date, settlement_date, counterparty_id, instrument_id, trade_type, quantity, price, notional_amount, currency, status) VALUES (:trade_date, :settlement_date, :counterparty_id, :instrument_id, :trade_type, :quantity, :price, :notional_amount, :currency, :status) WHERE :settlement_date >= :trade_date + INTERVAL '1 day'
  severity_level: HIGH
  source_tables: trades
  source_columns: trade_date, settlement_date
  unit: Date
  synonyms: Trade Settlement Date, Trade Date Plus One
  description: This rule ensures that the settlement date is at least one day after the trade date.

### [MEDIUM] VaR Threshold Alert
If var_95 > 1000000, flag position for review
SQL patterns:

  INSERT INTO risk_metrics (metric_id, position_id, metric_date, var_95, var_99, cvar, sharpe_ratio, max_drawdown, volatility) VALUES (:metric_id, :position_id, :metric_date, :var_95, :var_99, :cvar, :sharpe_ratio, :max_drawdown, :volatility) WHERE :var_95 > 1000000
  severity_level: MEDIUM
  source_tables: risk_metrics
  source_columns: var_95
  unit: Currency
  synonyms: VaR Threshold, Value-at-Risk Threshold
  description: This rule flags positions with a VaR greater than 1 million for review.
