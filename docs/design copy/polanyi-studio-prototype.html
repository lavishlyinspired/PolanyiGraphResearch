q2-counterparty-risk-policy.md

mentions highlighted · deterministic glossary scan guarantees term links

§4.2 — Desks must review aggregate exposure to any counterparty domiciled in a high-risk country on a weekly basis. Positions with a combined notional amount above EUR 25m require sign-off from the regional risk officer.

Under no circumstances may new positions be opened against a sanctioned entity; existing positions are to be unwound per the compliance schedule. Desks should additionally monitor wrong-way risk when collateral is issued by the counterparty itself.

Validator

Check any SQL against the declared rules. Deterministic verdict — no model involved. The same gate runs on every query the agent writes.

SQL

SELECT c.name, SUM(t.notional_amount) AS exposure
FROM trades t
JOIN counterparties c
  ON t.counterparty_id = c.counterparty_id
WHERE c.country_risk_rating >= 8
GROUP BY c.name
ORDER BY exposure DESC
CLI equivalent: polanyi validate "SELECT …"

Verdict

✕ BLOCKED
✕ BR-001
Sanctioned Counterparty Check
Query joins trades → counterparties but never references is_sanctioned. Add AND c.is_sanctioned = FALSE, or select sanctioned rows explicitly for compliance review.
! BR-004
USD Normalization
notional_amount is summed across currencies. Join fx_rates and convert, or use notional_usd.
✓ BR-002
Cancelled Trades Excluded from Revenue
Not a revenue aggregation — rule not applicable.
✓ BR-003
High-Risk Country Exposure Flag
Query filters on country_risk_rating — flag requirement satisfied.

SQL

SELECT c.name, SUM(t.notional_amount * fx.rate_to_usd) AS exposure_usd
FROM trades t
JOIN counterparties c
  ON t.counterparty_id = c.counterparty_id
 AND c.is_sanctioned = FALSE
JOIN fx_rates fx
  ON fx.currency = t.currency AND fx.rate_date = t.trade_date
WHERE c.country_risk_rating >= 8
GROUP BY c.name
ORDER BY exposure_usd DESC
